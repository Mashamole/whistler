The Whistler Bot

Download v1.0 (stable)

The Whistler bot is another jabber bot with MUC support, designed to be easy to extend, and it's built on the top of the SleekXMPP library.

Whistler is a generic framework to build MUC bots. The MUC rooms are the new IRC channels. Although exists many differences and, of course, many people still prefer the old and good IRC channel, the true is that the MUC rooms offers a number of new posibilities and utilities.

Whistler works using mixins. The mixins implements a bot personality, and of course you can mix a number of mixins. Sounds cool eh?!

Installation

Whistler is written in python, so you need almost a 2.7 version of python and the SleekXMPP library (which is installed as dependency).

Whistler has an entry in pypi, so you can install whistler using pip or setuptools:

$ pip install whistler
$ easy_install whistler

Usage

The bot is designed to work in foreground, you can use process monitoring tool to keep it running in background, like runit, daemontools or dmon.

To start the bot create a config file with required data, usually you need a server to connect to, a port, an username and a password, and of course a MUC channel to join in. The example config file is self explanatory.

To start the bot just type:

whistler config.file

Whistler is designed to be modular, once the bot is started without load any mixin, bot does nothing. Its a zombie without will.

Mixins give personality to the bot, so loading the log mixin converts the zombie bot into a logger bot; loading the help mixin gives the bot the ability of know itself and so on.

There are a list of mixins, but you can easy create new one.

To load a mixin just type in the config file an entry like that:

[mixin:mixin_name]
optional_properties

Extend

The real power of Whistler is the abaility to be extended. With mixins you can give to your bot a unique function or personality, and you can create easily new commands.

To create new mixin just create a new python file in mixins/ directory, and put there a new class (none parent class is required). Any method which starts with "cmd_" prefix will be threated as a bot command. For example just type a single mixin which give us the current date, so we create the file givemedate.py:

import datetime

class DateMixin(object):
    def cmd_givemedate(self, cmd, args):
        now = datetime.datetime.now()
        return now.strftime("%Y-%m-%d %H:%M")

Now just edit your config file and add the mixin:

[mixin:givemedate]

Now you can run the bot and enjoy your new command! That's easy, innit?

Also, bug reports/fixes, enhancements proposals, new mixins or, in general, any comment about the bot are welcome.
